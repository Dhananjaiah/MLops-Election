## Slide Template Guidelines

Each slide deck should follow this structure:

### Section Title Slide
- Section number and title
- Learning objectives
- Estimated time

### Content Slides
- One main concept per slide
- Minimal text, maximum visuals
- Code examples in large, readable fonts
- Architecture diagrams where applicable

### Lab/Demo Slides
- Clear step-by-step instructions
- Screenshots of expected output
- Common errors and fixes

### Summary Slide
- Key takeaways (3-5 bullets)
- Link to quiz
- Link to next section

## Design Guidelines

- **Font**: Clean sans-serif (Montserrat, Open Sans, Roboto)
- **Code Font**: Monospace (Fira Code, JetBrains Mono)
- **Colors**: Professional palette (blues, grays, accent color for highlights)
- **Images**: High-quality diagrams and screenshots
- **Animations**: Minimal, purposeful (for revealing steps)

## Tools Recommendations

1. **Google Slides** - Easy collaboration
2. **PowerPoint** - Rich features, offline
3. **Reveal.js** - Code-friendly, version controllable
4. **Canva** - Beautiful templates
5. **Figma** - Custom diagrams

## Export Format

- Export as PDF for distribution
- Keep source files (.pptx, .key) for editing
- Include speaker notes for instructors
